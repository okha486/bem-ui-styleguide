BEM UI Style Guide

BEM 방식과 웹접근성 기준을 바탕으로 정리한 실무형 UI 스타일가이드입니다. 재사용 가능한 컴포넌트 구조, 네이밍 규칙, 마크업 기준을 한 곳에서 확인할 수 있습니다.

가이드 개요

  • 구조화

    컴포넌트 단위로 UI를 분리해 재사용성과 유지보수성을 높입니다.

  • BEM 네이밍

    Block, Element, Modifier 규칙으로 일관된 클래스 체계를 관리합니다.

  • 웹접근성

    시맨틱 구조, 키보드 사용성, 대체텍스트, 레이블 연결을 고려합니다.

  • 실무 활용

    퍼블리싱 실무에서 바로 적용할 수 있도록 예시와 규칙을 함께 제공합니다.

기본 규칙

BEM 표기법

  • .block : 독립적으로 사용할 수 있는 컴포넌트
  • .block__element : block 내부를 구성하는 요소
  • .block--modifier : block의 상태 또는 변형
  • .block__element--modifier : element의 상태 또는 변형

웹접근성 기준

  • 링크와 버튼은 목적이 명확한 텍스트를 제공합니다.
  • 입력 요소는 label과 연결하고, 필요한 안내 문구를 함께 제공합니다.
  • 키보드만으로도 이동과 조작이 가능해야 합니다.
  • 아이콘 단독 버튼에는 대체 가능한 이름을 제공합니다.

문서 구성 원칙

  • 각 컴포넌트 문서는 제목, 설명, 예시, 상태, 접근성 유의사항 순서로 정리합니다.
  • 공통 스타일은 중복 선언하지 않고 재사용 가능한 구조로 관리합니다.
  • 시각 표현보다 의미 구조와 사용 규칙을 우선합니다.

컴포넌트 목록

각 컴포넌트 페이지에서 구조, 예시, 상태, 접근성 기준을 확인할 수 있습니다.

  • Typography

    텍스트 스타일, 제목, 본문, 강조 요소 기준
  • Navigation

    Skip Link, Breadcrumb, Step, 메뉴 구조
  • Layout

    레이아웃 기본 구조와 영역 분리 기준
  • Button

    기본 버튼, 아이콘 버튼, 정렬, 상태 예시
  • Form

    입력 요소, 선택 요소, 파일, 주소, 이메일 폼
  • Table

    데이터 테이블 구조와 캡션, 헤더, 정렬 기준
  • Tab

    탭 구조와 선택 상태, 키보드 접근 예시
  • List

    리스트형 UI, 설명 리스트, 정보 목록 패턴
  • Box

    박스형 UI, 검색영역
  • Etc

    기타 공통 UI와 보조 패턴 정리
